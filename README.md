# Novantra community

Welcome to the public community space for [Novantra](https://novantra.io), the governed operations platform for regulated industries.

This repository is where you file bug reports, request features, discuss the product, and give feedback on the documentation. The Novantra product itself is closed source; this repository is for community input only.

## Where to file what

| You want to... | Go here |
|---|---|
| Report a bug | [Open an Issue](https://github.com/novantra/community/issues/new/choose) with the Bug report template |
| Request a feature | [Open an Issue](https://github.com/novantra/community/issues/new/choose) with the Feature request template |
| Suggest a docs improvement | [Open an Issue](https://github.com/novantra/community/issues/new/choose) with the Docs feedback template |
| Ask a question or start a discussion | [Open a Discussion](https://github.com/novantra/community/discussions) |
| Report a security vulnerability | See [SECURITY.md](./SECURITY.md). Do **not** file a public Issue. |

If your bug is a production-impacting issue on a paid plan, contact your account team directly. This repository is best-effort and is not a support channel for incidents.

## Before opening an Issue

A few minutes of checking saves everyone time:

1. **Search existing Issues and Discussions** for the same topic. If you find one, add a reaction or comment rather than opening a duplicate.
2. **Check the [documentation](https://docs.novantra.io)** for whether the behavior is intentional or covered.
3. **Use the right template.** Blank Issues are disabled; the templates ask the questions we need answered to act.

## What this repository is *not*

- **Not the product source code.** The Novantra cloud platform and on-premise builds are closed source. We do not accept code pull requests because there is no code here to change.
- **Not a real-time support channel.** For urgent issues on paid plans, contact your account team. We respond here on a best-effort cadence.
- **Not a place for sensitive data.** Issues and Discussions are public. Never paste credentials, customer data, audit content, or any sensitive payload. Redact before posting.

## Pull requests

We have left pull requests technically enabled, but this repository has no code or content to change. PRs are closed with a pointer to the right place to file an Issue or Discussion. See [PULL_REQUEST_TEMPLATE.md](./.github/PULL_REQUEST_TEMPLATE.md).

## What you can expect

- **Acknowledgement** on Issues filed with a complete template: within a few business days.
- **Triage** (label + assignment to a future release or backlog): within two weeks.
- **Action** on accepted Issues: depends on scope and priority. Feature requests in particular are evaluated against the roadmap and may take time.

We do not promise that every Issue will be acted on, but we read all of them.

## Code of conduct

Participation in this repository is governed by our [Code of Conduct](./CODE_OF_CONDUCT.md). In short: be respectful, assume good faith, and keep discussion focused on the product.
